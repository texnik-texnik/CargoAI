# 🔧 Admin Panel - Групповое Обновление Треков

## 📋 Что Это

**Админ Панель** - это интерфейс для массового управления трек-кодами, который позволяет обновлять статусы сотен треков одной командой.

---

## 🚀 Как Это Работает

### Проблема (До):

```
У вас 300 новых трек-кодов в таблице
    ↓
Открываете каждую таблицу вручную
    ↓
Редактируете статус по одному
    ↓
300 таблиц × 1 минута = 5 часов! 😫
```

### Решение (После):

```
Загружаете таблицы в папку на Google Drive
    ↓
Отправляете одну команду: /bulk 19.03.2026_18.03.2026-received
    ↓
Бот автоматически обновляет все треки за 19-18 марта
    ↓
300 треков за 10 секунд! 🎉
```

---

## 📁 Шаг 1: Загрузка Трек-Кодов

### Структура Папки:

```
Google Drive/
└── Трек-Коды (ID в Script Properties)
    ├── 2026-03-19.xlsx (300 треков)
    ├── 2026-03-18.xlsx (250 треков)
    └── 2026-03-17.xlsx (280 треков)
```

### Как Загружать:

1. **Получаете таблицу** от партнёров (Excel/CSV)
2. **Конвертируете** в Google Sheets (автоматически при загрузке)
3. **Загружаете** в папку "Трек-Коды" на Google Drive
4. **Готово!** Бот автоматически видит новые файлы

---

## 🔧 Шаг 2: Групповое Обновление

### Формат Команды:

```
/bulk DD.MM.YYYY_DD.MM.YYYY-status
```

### Примеры:

#### 1. Обновить треки за 19 марта в статус "В пути":
```
/bulk 19.03.2026_19.03.2026-intransit
```

#### 2. Обновить треки с 18 по 19 марта в статус "На складе":
```
/bulk 18.03.2026_19.03.2026-warehouse
```

#### 3. Обновить все треки за неделю:
```
/bulk 12.03.2026_19.03.2026-received
```

---

## 📊 Доступные Статусы

| Статус | Описание | Когда Использовать |
|--------|----------|-------------------|
| `waiting` | ⏳ Ожидается | Трек создан, но не отправлен |
| `received` | ✅ Принят на склад | Получен от отправителя |
| `intransit` | 🚚 В пути | Между складами |
| `border` | 👮 На таможне | Проходит таможню |
| `warehouse` | 🏭 На складе | Прибыл на склад в Хуросоне |
| `payment` | 💰 Оплачен | Клиент оплатил получение |
| `delivered` | 🎉 Выдан | Клиент получил груз |

## ⚠️ Возможные Ошибки

### Ошибка 1: "Invalid date format"

**Причина:** Неправильный формат даты

**Решение:**
```
❌ 19.3.2026_18.3.2026-received
✅ 19.03.2026_18.03.2026-received
```

Всегда используйте 2 цифры для дня и месяца!

---

### Ошибка 2: "Invalid status"

**Причина:** Несуществующий статус

**Решение:**
```
❌ 19.03.2026_18.03.2026-ontheway
✅ 19.03.2026_18.03.2026-intransit
```

Используйте только валидные статусы из списка!

---

### Ошибка 3: "Access denied"

**Причина:** Ваш ID не в ADMIN_IDS

**Решение:**
```
1. Открыть Script Properties
2. Найти ADMIN_IDS
3. Добавить ваш ID через запятую
4. Save
```

---

### Ошибка 4: "Updated 0 tracks"

**Причина:** Нет треков в указанном диапазоне дат

**Решение:**
```
1. Проверить даты в таблицах
2. Убедиться, что формат дат совпадает
3. Попробовать другой диапазон
```
