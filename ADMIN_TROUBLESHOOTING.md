# 🔧 Admin Panel Troubleshooting

## Проблема: /admin не работает

### Причина 1: Команда не добавлена в Main.gs ✅ ИСПРАВЛЕНО

**Решение:**
```
Команда /admin добавлена в Main.gs
Изменения запушены: clasp push -f
```

**Проверка:**
```
1. Открыть бота в Telegram
2. Отправить: /admin
3. Если вы админ → Покажет кнопку
4. Если не админ → Игнорирует
```

---

### Причина 2: Ваш ID не в ADMIN_IDS

**Проверка:**
```
1. Открыть: https://script.google.com/home/projects/YOUR_ID/edit
2. Нажать: ⚙️ Settings → Script Properties
3. Найти: ADMIN_IDS
4. Проверить: Ваш ID там есть?
```

**Как узнать свой ID:**
```
1. Открыть Telegram
2. Отправить боту: /start
3. Посмотреть логи (Google Sheets → Logs)
4. Найти ваш User ID
```

**Или через бота:**
```
Отправить: /id
Бот ответит: Ваш ID: 123456789
```

**Добавить ID:**
```
1. Script Properties → ADMIN_IDS
2. Добавить: 123456789,987654321
3. Save
```

---

### Причина 3: Бот не обновился

**Проверка:**
```
1. Открыть: https://script.google.com/home/projects/YOUR_ID/executions
2. Посмотреть последние выполнения
3. Если ошибок нет → Всё OK
```

**Обновить:**
```bash
cd /storage/emulated/0/CargoBot/miniapp
clasp push -f
```

## Проблема: Кнопка не открывается

### Причина 1: Web App не развёрнут

**Решение:**
```
1. Открыть: https://script.google.com/home/projects/YOUR_ID/edit
2. Нажать: Deploy → New Deployment
3. Type: Web App
4. Execute as: Me
5. Who has access: Anyone
6. Deploy
7. Скопировать URL
```

### Причина 2: Неверный URL в AdminCommands.gs

**Проверка:**
```javascript
// AdminCommands.gs
const webAppUrl = doGetAdminPanel({ parameter: { uid: userId } }).getUrl();
```

Функция автоматически генерирует URL, но можно указать вручную:

```javascript
const webAppUrl = "https://script.google.com/macros/s/YOUR_DEPLOYMENT/exec";
```

## Проблема: /bulk не работает

### Причина 1: Неправильный формат

**Правильный формат:**
```
✅ /bulk 19.03.2026_19.03.2026-received
❌ /bulk 19.3.2026_19.3.2026-received
❌ /bulk 19.03.2026-19.03.2026-received
❌ /bulk 19.03.2026_19.03.2026_received
```

**Требования:**
- 2 цифры для дня (19, не 9)
- 2 цифры для месяца (03, не 3)
- Подчёркивание между датами (_)
- Тире перед статусом (-)
- Статус на английском (received, не получен)

### Причина 2: Нет треков в диапазоне

**Проверка:**
```
1. Открыть Google Drive → Папка с треками
2. Открыть таблицу
3. Проверить колонку с датой
4. Убедиться, что даты совпадают
```

## Тестирование

### Тест 1: Проверка Админа

```
1. Отправить: /admin
2. Ожидать: Кнопка "🔧 Открыть Админ Панель"
3. Если нет → Проверить ADMIN_IDS
```

### Тест 2: Проверка Команды

```
1. Загрузить тестовую таблицу (5 треков)
2. Отправить: /bulk 17.03.2026_17.03.2026-received
3. Ожидать: "✅ Обновлено: 5"
4. Проверить таблицу → Статусы изменились
```

### Тест 3: Проверка Панели

```
1. Отправить: /admin
2. Нажать: "🔧 Открыть Админ Панель"
3. Ожидать: Откроется Web App
4. Выбрать даты → Обновить
5. Ожидать: "✅ Обновлено: X"
```
